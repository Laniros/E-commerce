Specify all the different route your app supports


authRoutes
post - http://localhost:5000/api/signup - a request to sign a new user to the store users DB
post - http://localhost:5000/api/signin - a request to sign In a user already signup to the store
get - http://localhost:5000/api/signout - signing out of the store – just temporary exit

braintreeRoutes

get - http://localhost:5000/api/braintree/getToken/:userId – a request to get a token from Braintree sandbox API in order to later present the features given by BRAINTREE.
post - http://localhost:5000/api/braintree/payment/:userId – sent a payment to the Braintree sandbox.

userRoutes
get - http://localhost:5000/api/secret/:userId – get the user profile ( this for getting the token later on)
get - http://localhost:5000/api/user/:userId – get the user information
get - http://localhost:5000/api/orders/by/user/:userId – get user purchase history.
put - http://localhost:5000/api/user/:userId - update the user information ( usually password)

categoryRoutes
get - http://localhost:5000/api/category/:categoryId – get one category information
post - http://localhost:5000/api/category/create/:userId - create one category make sure that the user is admin.
put - http://localhost:5000/api/category/:categoryId/:userId" - update one category makes sure that the user is admin.
delete - http://localhost:5000/api/category/:categoryId/:userId – delete one category make sure that the user is admin.
get - http://localhost:5000/api/categories - get all categories

productRoutes
get - http://localhost:5000/api/products - get all products
get - http://localhost:5000/api/products/search – get the searched products
get - http://localhost:5000/api/products/related/:productId – get all the products related to this game.
get - http://localhost:5000/api/products/categories – get the products according to the filtered categories.
get - http://localhost:5000/api/products/by/search – get the products according to the filtered price range.
post - http://localhost:5000/api/product/photo/:productId – get the product photo

orderRoutes
post - http://localhost:5000/api/order/create/:userId – a request when a user makes a payment and creates order.
get - http://localhost:5000/api/order/list/:userId – get the list of all orders that the user have
get - http://localhost:5000/api/order/status-values/:userId – get the status value of an order ( shipping, process and ex…)
put - http://localhost:5000/api/order/:orderId/status/:userId - update the status value of an order ( shipping, process and ex…)

How did you make your store secured?

1. Each user can do his operations on his user only – required signUp and signIn for each user

2. The passwords is encrypted and not presented or saved anywhere- just the encrypted- no way do get it from the DB or the request.

3. We used express-rate-limit on order to prevent DDOS attacks.
